## 2. 起動時間（受け入れ基準1: 3 秒以内）

- [ ] アプリアイコンタップ → タイトル画面（「はじめから」ボタンが押せる状態）の
      表示まで **3 秒以内**であること。
- 計測方法:
  1. デバイスを 30 秒間アイドル状態にしてキャッシュをクリア。
  2. アプリを **完全終了**（タスクキル）してから起動。
  3. アイコンタップとほぼ同時にストップウォッチを開始。
  4. タイトル画面の「はじめから」ボタンが表示された瞬間に停止。
  5. 3 回計測して中央値を採用。
- 設計上の最適化:
  - `lib/main.dart` の起動シーケンスで Settings → Save/Archive の 2 段階
    （Save と Archive は `Future.wait` で並行）。
  - `LoggingAudioService(keepHistory: false)` で履歴蓄積を抑制。
  - 旧実装の「3 ロード直列 await」は廃止済み。

## 3. メモリ（受け入れ基準2: 2 倍以内）

- [ ] アプリ起動直後のメモリ使用量を記録（A）。
- [ ] 30 分プレイ後のメモリ使用量を記録（B）。
- [ ] 1 時間プレイ後のメモリ使用量を記録（C）。
- [ ] **C ≤ A × 2** を満たすこと。
- 計測方法:
  - Flutter DevTools の Memory タブで「Heap」「External」を確認。
  - Android Studio Profiler では「Java/Kotlin Heap」「Native Heap」「Graphics」
    「Code」「Stack」を合算した値を比較。
  - 1 時間の間に「画面遷移」「会話イベント」「セーブ/ロード」「設定変更」
    「ショップ購入」を均等に実行する。
- 既知のリーク候補（Sprint 12 で修正済み）:
  - `DialogueModal` / `EventPlayer` の dispose 時に `AppScope.of(context)`
    を呼ぶ箇所 → State 側に `AudioService?` をキャッシュする方式に変更。
  - `AffinityHearts` の didUpdateWidget で都度 try/catch していた
    `AppScope.of(context)` → didChangeDependencies でキャッシュ。
  - `HomeScreen._triggerEventFlash` の `Future.delayed` → `Timer` 化し dispose
    で cancel。
  - `CgRevealScreen` の `Future.delayed` 自動クローズ → `Timer` 化し
    dispose で cancel。
  - `LoggingAudioService` の history list 無限増殖 → 本番は
    `keepHistory: false`。
  - `GameState.dispose` で `Inventory.dispose` および各リスナリスト
    （`_dayAdvanceListeners` / `_statChangeListeners`）の `clear` を追加。
