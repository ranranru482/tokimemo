# 月と珈琲 (tokimemo)

社会人 1 年生として 1 年間を生き、5 人のキャラクターと出会い、自分なりの結末にたどり着く Flutter 製のオリジナル恋愛シミュレーション。

> 既存 IP（ときめきメモリアル等）の固有名詞・台詞は一切使用していない完全オリジナル作品。社会人らしい抑制の効いた大人びたトーンで統一。

---

## ゲーム概要

- 主人公は新卒入社の社会人。プレイ期間は 4 月 1 日から翌年 3 月 31 日までの 1 年間。
- 1 日は朝 / 日中 / 夕方 / 夜の 4 枠で進行。平日日中は「仕事」固定、それ以外は読書・運動・外出・誘いなどの行動を選択する。
- 5 人のキャラクター（七瀬 灯 / 久遠 詩 / 鴻巣 透 / 蓮見 紗夜 / 槙原 結衣）と出会い、表面好感度と真の好感度の 2 層で関係を深めていく。
- 3 月 31 日の年度末で、プレイ内容に応じた 1 つのエンディングが再生される。

---

## 主な機能

### 基本ループ
- 4 枠タイムラインと能力値（体力 / 所持金 / ストレス / 知性 / 感性 / 社交 / 仕事評価）管理。
- 月初給料、週次ふりかえり、月カレンダー予約、ショップでの贈り物購入。
- 平日日中は仕事ミニ判定（成功率 = 仕事評価ベース）。**35% の確率で仕事中イベントが発生**し、選択肢で評価・ストレス・所持金・好感度が動く。

### キャラクター・イベント
- 出会いイベント 5 本 / 個別イベント **35 本**（各キャラ 7 本） / 告白前夜 **5 本**。
- 共通イベント 10 本（新年度の朝・健康診断・夏祭り・クリスマス・バレンタイン 等）+ ランダム遭遇 8 本。
- 仕事中イベント **7 本** × 各 2 選択肢（上司 / 同僚 / プロジェクト / ミス / チャンス）。
- 全イベントに表面好感度 / 真の好感度 / ストレスへの差分を持つ選択肢を内包。

### エンディング（9 種）
- **バッドED 2 種**: 燃え尽きED（ストレス ≥90）/ 左遷ED（仕事評価 ≤10）— 最優先で判定。
- 個別ED 5 種: 表面 80 + 真 20 + 告白前夜通過の AND で発火。
- ノーマルED 1 種 / 真ED「月と珈琲ED」1 種（全キャラ表面 60 + 真 30 + ストレス ≤40 + 仕事評価 ≥50 + CG 12 件以上）。

### システム
- セーブ / ロード（手動 10 + クイック 1 + オート 3 のリング）+ 月初・週末・節目前の自動セーブ。
- メモリーアルバム（CG 56 件、カテゴリ / キャラフィルタ + 未解放ヒント表示）。
- エンディング図鑑（バッド / 個別 / ノーマル / 真の 4 セクション、達成数カウンタ、未達成ヒント 3 行）。
- 設定（テーマモード、BGM / SE 音量、テキスト速度）と SharedPreferences 永続化。
- BGM / SE / ボイスフィールドを論理キーで設計（実音は audioplayers ベース、アセット未投入時は無音で握りつぶし）。

## 開発方法

本プロジェクトは仕様書ベースのスプリント駆動で開発している。

1. **Planner**: `docs/spec.md` を読み、機能とスプリントを定義する。
2. **Generator**: スプリントを 1 つずつ実装し、自己評価する。
3. **Evaluator**: 実装結果をテスト・実機で検証し、合否を判定する。

各スプリント完了時に `docs/progress.md` の末尾に追記する。実装の方針として:

- 既存セーブ互換を壊さない（`GameState.toMap` / `restoreFromMap` のスキーマ後方互換）。
- 既存テストを壊さない（追加は新規ファイル、改修は最小差分）。
- 大規模リファクタを禁止し、最小差分で機能を足す。
- 実画像 / 実音は `CgView` / `CharacterPortrait` / `ScenicBackground` / `LoggingAudioService` のプレースホルダで開発を進める。

## Flutter 起動方法

### 必須環境
- Flutter SDK 3.x 以降（`pubspec.yaml` の制約参照）。
- Android Studio または Xcode（実機 / エミュレータ用）。

### セットアップ
```sh
flutter pub get
```

### 実行

#### Android エミュレータ
```sh
flutter emulators --launch <emulator_id>
flutter run -d emulator-5554
```

#### Windows デスクトップ
```sh
flutter run -d windows
```

### テスト
```sh
# 全 unit / widget テスト
flutter test

# 静的解析
flutter analyze

# integration test（要デバイス接続）
flutter test integration_test/ -d emulator-5554
```

### ビルド
```sh
# Android デバッグ APK
flutter build apk --debug

# Android リリース APK
flutter build apk --release
```
